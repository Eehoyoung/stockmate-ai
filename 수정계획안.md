## 1. 이미 수정된 항목 (2026-04-01 완료)

| # | 파일 | 수정 내용 | 근거 |
|---|------|---------|------|
| 1 | `WebClientConfig.java` | `defaultHeader(Content-Type: application/json)` 제거 | WebClient `defaultHeader` + 요청별 `header()` 중복 → 두 개의 `Content-Type` 전송됨. 키움 서버 HTTP 200 래핑 500 원인 |
| 2 | `KiwoomApiRequest.java` | `stexTp` 필드에 `@Builder.Default` 추가 | `@SuperBuilder` 빌더 경로에서 DTO 필드 초기값(`= "1"`)이 무시됨 → `stex_tp` null → JSON 제외 → 1511 에러 (ka10032 확인) |

## 2. 핵심 배경 지식 — `@SuperBuilder` 함정

```
@SuperBuilder 로 빌더를 사용할 경우, DTO 클래스의 필드 초기값(= "xxx")은
new Foo() 생성자 방식에서만 적용됩니다.
.builder()...build() 경로에서는 명시적으로 .fieldName(value)를 호출하지 않으면
해당 필드가 null 이 됩니다.

@JsonInclude(NON_NULL) 가 적용되어 있으므로 null 필드는 JSON에서 제외됩니다.
→ 키움 API 필수 파라미터가 누락되면 1511 에러("필수 입력 값 없음") 발생.

해결책:
  A) @Builder.Default 어노테이션 추가 (부모·자식 클래스 모두 적용 가능)
  B) 빌더 호출부에서 명시적으로 값을 설정
```

## 3. 미수정 이슈 목록

### P1 — 기능 오류 (즉시 수정 필요)

#### 3-1. `NtlPricRequest` (ka10016) — S10 전략 작동 불가

**파일**: `CandidateService.java` `getS10Candidates()` 약 357번째 줄

**현재 코드**:
```java
NtlPricRequest.builder()
    .mrktTp(market)
    .build()   // ← mrktTp만 설정
```

**문제**: ka10016 필수 파라미터(Required=Y) 9개 중 7개가 빌더에서 설정되지 않음 → null → JSON 제외 → API 1511 오류

| 파라미터 | Required | DTO 초기값 | 빌더 호출 | 전송 여부 |
|---------|---------|-----------|---------|---------|
| `mrkt_tp` | Y | "000" | `.mrktTp(market)` ✓ | O |
| `ntl_tp` | Y | "1" | 없음 | **X (null)** |
| `high_low_close_tp` | Y | "1" | 없음 | **X (null)** |
| `stk_cnd` | Y | "1" | 없음 | **X (null)** |
| `trde_qty_tp` | Y | "00010" | 없음 | **X (null)** |
| `crd_cnd` | Y | "0" | 없음 | **X (null)** |
| `updown_incls` | Y | "0" | 없음 | **X (null)** |
| `dt` | Y | "250" | 없음 | **X (null)** |
| `stex_tp` | Y | — | @Builder.Default ✓ | O |

**영향**: S10 전략(`strategy_10_new_high.py`)의 Java 측 후보 풀(`candidates:s10:{market}`)이 항상 빈 리스트 반환 → S10 전략 신호 없음

**수정 방법**:
`NtlPricRequest` 클래스의 모든 필드에 `@Builder.Default` 추가, 또는 빌더 호출에 누락 파라미터를 명시:
```java
// 방법 A: NtlPricRequest 클래스 수정 (모든 필드에 @Builder.Default 추가)
@Builder.Default @JsonProperty("ntl_tp") private String ntlTp = "1";
@Builder.Default @JsonProperty("high_low_close_tp") private String highLowCloseTp = "1";
@Builder.Default @JsonProperty("stk_cnd") private String stkCnd = "1";
@Builder.Default @JsonProperty("trde_qty_tp") private String trdeQtyTp = "00010";
@Builder.Default @JsonProperty("crd_cnd") private String crdCnd = "0";
@Builder.Default @JsonProperty("updown_incls") private String updownIncls = "0";
@Builder.Default @JsonProperty("dt") private String dt = "250";

// 방법 B: 빌더 호출부에 명시 (CandidateService)
NtlPricRequest.builder()
    .mrktTp(market)
    .ntlTp("1")
    .highLowCloseTp("1")
    .stkCnd("1")
    .trdeQtyTp("00010")
    .crdCnd("0")
    .updownIncls("0")
    .dt("250")
    .build()
```
**권장**: 방법 A (`StrategyRequests.java` 수정) — DTO가 의도한 기본값을 보장하고 호출부를 간결하게 유지

### P2 — 잠재적 버그 (단기 수정 권장)

#### 3-2. `IntradayInvestorRequest` — `stexTp` 필드 shadow

**파일**: `StrategyRequests.java` `IntradayInvestorRequest` 클래스

**현재 코드**:
```java
public static class IntradayInvestorRequest extends KiwoomApiRequest {
    // ...
    @JsonProperty("stex_tp") private String stexTp = "1";  // ← 부모와 동일 이름으로 재선언
}
```

**문제**: 부모 `KiwoomApiRequest`의 `@Builder.Default stexTp = "1"`을 자식 필드가 shadow.
자식 필드는 `@Builder.Default` 없이 `= "1"` 초기값만 있으므로, 빌더 경로에서 **null** 이 됨.

**현재 영향**: `ka10063`은 `KiwoomApiService`에 편의 메서드가 없고 현재 미사용 상태이므로 실제 오류는 발생하지 않음.
**위험**: 향후 ka10063(장중투자자별매매) 구현 시 stex_tp 누락으로 1511 오류.

**수정 방법**:
```java
// IntradayInvestorRequest에서 stexTp 필드 제거 (부모 @Builder.Default로 관리)
public static class IntradayInvestorRequest extends KiwoomApiRequest {
    @JsonProperty("mrkt_tp") private String mrktTp = "000";
    @JsonProperty("amt_qty_tp") private String amtQtyTp = "1";
    @JsonProperty("invsr") private String invsr = "6";
    @JsonProperty("frgn_all") private String frgnAll = "1";
    @JsonProperty("smtm_netprps_tp") private String smtmNetprpsTp = "1";
    // stexTp 제거 — KiwoomApiRequest @Builder.Default 상속
}
```

#### 3-3. `IntradayInvestorUpperRequest` — 필드 초기값 @Builder.Default 누락

**파일**: `StrategyRequests.java` `IntradayInvestorUpperRequest`

**현재 코드**:
```java
@JsonProperty("trde_tp") private String trdeTp = "1";    // @Builder.Default 없음
@JsonProperty("mrkt_tp") private String mrktTp = "000";  // @Builder.Default 없음
@JsonProperty("orgn_tp") private String orgnTp = "9999"; // @Builder.Default 없음
```

**문제**: 빌더 사용 시 모든 필드가 null → JSON 제외 → ka10065 호출 시 필수 파라미터 누락 가능
**현재 영향**: ka10065 현재 미사용. 향후 사용 시 오류.

| 헤더 | 설정값 | 상태 |
