## 1. 공통 변경 사항

이번 개편에서 전 전략에 공통으로 반영된 핵심은 아래 5가지다.

1. `tp_sl_engine.py`가 전략별 TP/SL의 최종 단일 소스가 되도록 강화했다.
2. 전략별 `min_rr`를 차등 적용한다.
   - 데이 전략: `1.6`
   - 단기 스윙: `1.45`
   - 추세/돌파 스윙: `1.55`
3. 전략별 `time_stop_type / time_stop_minutes / time_stop_session` 메타를 추가했다.
4. `position_monitor.py`에 `TIME_STOP` 청산을 추가했다.
5. 엔진에서 계산한 `rr_ratio / trailing / strategy_version / time_stop` 메타가 DB와 포지션 모니터까지 이어지도록 저장 경로를 연결했다.

## 2. 전략별 변경 요약

### S1_GAP_OPEN

- 기존 핵심: 갭 유지 실패 시 `prev_close × 0.995` 손절
- 이번 변경:
  - `min_rr`를 데이 전략 기준 `1.6`으로 강화
  - 시간청산 추가: `intraday_minutes=60`
  - 세션 정리 추가: `same_day_close`
- 의미:
  - 갭 전략을 당일 초반 모멘텀 전략으로 명확히 고정
  - 안 가는 종목을 오래 들고 가지 않도록 변경

### S2_VI_PULLBACK

- 기존 핵심: ATR 1.0 기반 타이트 손절, `vi_price` 복귀 목표
- 이번 변경:
  - `min_rr=1.6`
  - 시간청산 추가: `intraday_minutes=30`
  - 세션 정리 추가: `same_day_close`
- 의미:
  - VI 눌림목의 본질인 "빠른 재상승 실패"를 시간으로 처리

### S3_INST_FRGN

- 기존 핵심: MA20 / 스윙저점 / fib 기반 스윙 TP/SL
- 이번 변경:
  - 전략 분류를 day 쪽이 아니라 swing 쪽으로 이동
  - `min_rr=1.45`
  - 시간청산 추가: `trading_days=2`
- 의미:
  - 실제 산식이 2~3일 스윙형인데 시스템 분류가 데이였던 충돌을 정리

### S4_BIG_CANDLE

- 의미:

### S5_PROG_FRGN

## 4. 사용자 매도 추천 알림 설정 여부

결론: **설정되어 있음**

### 4-1. Python 포지션 모니터에서 매도 추천 발행

`ai-engine/position_monitor.py`가 아래 2종류의 알림을 발행한다.

- `SELL_RECOMMENDATION`
- `SELL_SIGNAL`

발행 채널:
- 기본값 `ai_scored_queue`

반영된 청산 사유:
- `SL_HIT`
- `TP1_HIT`
- `TP2_HIT`
- `TIME_STOP`
- `TRAILING_STOP`
- `TREND_REVERSAL`

### 4-2. TIME_STOP도 사용자 알림에 반영됨

이번 변경으로 `TIME_STOP`에 대해 아래가 연결되어 있다.

- `position_monitor.py`에서 `SELL_RECOMMENDATION` 발행
- `position_monitor.py`에서 `SELL_SIGNAL` 발행
- `telegram-bot/src/utils/formatter.js`에서 `TIME_STOP` 이유 표시
- `telegram-bot/src/handlers/confirmGate.js`에서 time stop 정책 표시

즉, 사용자는 시간손절 사유도 메시지로 받을 수 있도록 설정되어 있다.

### 4-3. Java 포지션 모니터 충돌 방지

`api-orchestrator/.../PositionMonitorScheduler.java`는 기본적으로 비활성화되도록 변경했다.

- 환경변수: `JAVA_POSITION_MONITOR_ENABLED`
- 기본값: `false`

의도:
- Python `position_monitor.py`를 실전 청산 기준으로 단일화
- Java와 Python이 동시에 같은 포지션을 청산하는 충돌 방지

## 6. 최종 결론

이번 반영으로 S1~S15 전체 전략은 단순 가격 TP/SL만 쓰는 구조에서 아래 구조로 강화됐다.

- 전략별 차등 `min_rr`
- 전략별 `time stop`
- TP1 이후 trailing 메타 저장
- Python 포지션 모니터 기준의 `TIME_STOP` 실제 청산
- 사용자 대상 `SELL_RECOMMENDATION / SELL_SIGNAL` 알림 연동

즉, 현재 상태는

1. 각 전략별 TP/SL 변경이 코드에 반영되어 있고  
2. 사용자에게 매도 추천 알림이 가도록 설정되어 있으며  
3. 핵심 테스트와 컴파일이 통과한 상태다.
