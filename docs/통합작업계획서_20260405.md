## 1. P1-A: `strategy_12_closing.py` — candidates:s12:* 풀 우선 읽기

### 배경

`candidates_builder._build_s12()`이 `candidates:s12:{market}` 키를 정상 생성하고 있으나, `strategy_12_closing.py`의 `scan_closing_buy()`는 항상 ka10027 + ka10063을 직접 호출한다. S12는 14:30~14:50 타이밍이 중요하므로 API 호출 지연을 줄이기 위해 풀 우선 읽기를 적용해야 한다.

### 변경 범위

**파일**: `ai-engine/strategy_12_closing.py`

**변경 전**: `scan_closing_buy()` — ka10027 직접 호출 → 전체 목록 스캔

**변경 후**: candidates:s12:* 풀 → 교차 필터 → ka10027 직접 fallback

```python
async def scan_closing_buy(token: str, market: str = "000", rdb=None) -> list:
    # 1. candidates:s12:{market} 풀 읽기
    pool_set: set[str] = set()
    if rdb:
        try:
            pool = await rdb.lrange(f"candidates:s12:{market}", 0, -1)
            pool_set = set(pool)
            if pool_set:
                logger.debug("[S12] 풀 %d종목 로드 (candidates:s12:%s)", len(pool_set), market)
        except Exception as e:
            logger.warning("[S12] 풀 조회 실패 (fallback): %s", e)

    # 2. 기관 순매수 + 등락률 상위 병렬 조회
    gainers_task = fetch_top_gainers_paged(token, market)
    inst_set_task = fetch_inst_netbuy_set(token, market)
    gainers, inst_set = await asyncio.gather(gainers_task, inst_set_task)

    results = []
    for item in gainers:
        stk_cd = item.get("stk_cd")
        if not stk_cd or stk_cd not in inst_set:
            continue
        # 풀이 있을 경우 풀 내 종목만 처리 (API 응답 최소화)
        if pool_set and stk_cd not in pool_set:
            continue
        # ... 기존 조건 검증 로직 유지 ...
```

### 검증 포인트

- 풀이 비어있을 때(장 초기, TTL 만료) fallback 동작 확인
- `market="000"` (전체) 호출 시 pool_set 로직 처리 (001+101 합산 필요)

### 변경 범위

**파일**: `ai-engine/scorer.py` — `_time_bonus()` 함수 내

### 트레이더 관점: 올바른 TP/SL 설정 철학

> "SL은 '내 가설이 틀렸다고 확인되는 가격'이다. MA가 무너지면 추세 가설이 틀린 것이다. 그 아래에 설정하면 된다. 임의의 -5%는 의미가 없다." — 트레이더 페르소나

#### 원칙 1: SL = "가설 무효화 가격"

| 전략 | 진입 근거 (가설) | 가설 무효화 기준 (SL) |
|------|----------------|---------------------|
| S8 골든크로스 | MA5 > MA20 크로스 = 추세 전환 | MA20 이탈 (크로스 실패 확인) |
| S9 눌림목 | MA5 지지에서 반등 | MA5 이탈 + 1ATR 아래 (지지 붕괴) |
| S10 신고가 | 전 고점 돌파 = 공급 소화 | 돌파 레벨 아래 복귀 (Fake Breakout) |
| S13 박스돌파 | 박스 상단 돌파 | 박스 상단선 아래 (돌파 실패) |
| S14 과매도 | RSI/ATR 수렴 = 반등 에너지 축적 | 직전 스윙 저점 아래 (저점 갱신) |
| S1 갭상승 | 갭이 추세 신호 | 갭 메워짐 (시가 아래) |
| S7 동시호가 | 장전 강한 매수세 | 시초가 아래 (강도 확인 실패) |

#### 원칙 2: TP = "다음 명확한 공급 구간"

```
저항 탐색 우선순위:

  1순위: 차트 구조적 저항
     ├─ 최근 60봉 내 스윙 고점 클러스터 (2% 이내 2개 이상)
     ├─ 직전 고점 (심리적 공급 집중 지점)
     └─ 박스권 상단 (S13은 박스 높이만큼 투영)

  2순위: 피보나치 확장 (돌파·추세 전략)
     ├─ 최근 스윙 저점 → 고점 범위 기준
     └─ 1.000 / 1.272 / 1.618 확장선 중 가장 가까운 것

  3순위: 볼린저 상단 (통계적 과매수 한계, 평균 복귀 압력 존재)

  4순위: ATR 배수 (구조적 저항 부재 시 — 신고가 공백 구간 등)
     ├─ 데이트레이딩: ATR × 2.0~3.0
     └─ 스윙: ATR × 3.0~5.0

  최종 검증: R:R = (TP - 진입가) / (진입가 - SL) ≥ 1.3
     └─ R:R < 1.3이면 → TP를 (진입가 - SL) × 1.3 으로 강제 상향
```

#### 원칙 3: 변동성 상태에 따른 배수 조정

ATR%는 종목의 "일상적 움직임 크기"다. SL을 ATR 배수로 설정할 때 변동성이 높은 종목은 배수를 줄이고, 낮은 종목은 늘려야 한다.

```
ATR% 구간별 SL 배수 (스윙 전략):

  ATR% < 1.0% (저변동성)   : ATR × 2.5  (노이즈가 작으므로 타이트하게 가능)
  1.0% ≤ ATR% < 2.0%       : ATR × 2.0  (표준)
  2.0% ≤ ATR% < 3.5%       : ATR × 1.7  (변동성 높으니 배수 줄여 손실 제어)
  ATR% ≥ 3.5% (고변동성)   : ATR × 1.5  (패닉 구간 — 진입 자체 재검토)
```

### 전략별 TP/SL 상세 설계

#### 그룹 A: 스윙 전략 (S8~S15) — 추가 API 호출 0회

이미 `candles` (일봉 60봉 이상)를 조회하므로 ATR, MA, 스윙 고저점 모두 재계산 가능.

| 전략 | SL 방법 | TP 방법 | 비고 |
|------|---------|---------|------|
| **S8 골든크로스** | MA20 아래 (크로스 무효화) | 볼린저 상단 or 직전 고점 | MA20 < SL_min 시 ATR×2.0 |
| **S9 눌림목** | MA5 아래 + ATR×0.3 (지지 붕괴) | 직전 스윙 고점 or 볼린저 상단 | |
| **S10 신고가** | 돌파 레벨(직전 고점) 아래 ATR×0.5 | 피보나치 1.272~1.618 확장 | 공백 구간이므로 피보나치 우선 |
| **S11 외인연속** | MA20 아래 (추세 지지 무효) | 볼린저 상단 or ATR×3.5 | |
| **S12 종가강도** | 당일 저가 아래 or ATR×1.5 | 직전 고점 or ATR×2.5 | 짧은 보유기간 반영 |
| **S13 박스돌파** | 박스 상단선 아래 (돌파 실패 확인) | 박스 높이 투영 (box_height) | 가장 명확한 구조적 TP |
| **S14 과매도** | 직전 스윙 저점 아래 | 볼린저 중심선(MA20) → 상단 | 이미 ATR 기반 — MA 지지로 고도화 |
| **S15 모멘텀** | MA20 아래 ATR×0.5 | 볼린저 상단 or 직전 고점 | TP 현재 +12% 하드코딩 → 동적화 |

**S13 박스권 돌파 TP 계산 예시 (가장 우아한 사례):**

```python
# 박스 높이 = 최근 15봉 최고가 - 최저가
box_high = max(highs[:15])
box_low  = min(lows[:15])
box_height = box_high - box_low

# 돌파 후 TP = 돌파 레벨 + 박스 높이 (박스 투영)
target_price = box_high + box_height

# SL = 박스 상단 아래 복귀 = 돌파 실패 확인
stop_price = box_high - atr * 0.5
```

이것이 트레이더들이 실제로 사용하는 "박스 투영 목표" 기법이다.

#### 그룹 B: 데이트레이딩 (S1~S7) — 분봉 추가 조회 필요

데이트레이딩은 보유기간이 당일이므로 일봉 ATR이 아닌 단기 변동성이 필요하다.

**옵션 1: 일봉 ATR 스케일링 (추가 API 없음)**

```python
# 일봉 ATR를 분봉 단위로 스케일링
# 거래 분(390분/일 기준), ATR 시간 비례
intraday_atr_est = atr_daily * (holding_minutes / 390) ** 0.5
```

이 방법은 통계적으로 합리적이나 근사치이며, 실제 장중 변동성을 반영하지 못한다.

**옵션 2: 분봉 ATR 직접 조회 (ka10080, 종목당 1회)**

```python
from indicator_rsi import fetch_minute_candles
min_candles = await fetch_minute_candles(token, stk_cd, tic_scope="5")
# 5분봉 ATR(14) → 장중 실변동성 측정
min_highs  = [c.get("high") for c in min_candles]
min_lows   = [c.get("low") for c in min_candles]
min_closes = [c.get("close") for c in min_candles]
atr_5min   = calc_atr(min_highs, min_lows, min_closes, 14)[0]
```

**권고: 옵션 2 (분봉 ATR) + 일봉 ATR 상한 조합**

```python
# 분봉 ATR이 일봉 ATR의 60% 초과 시 → 이상 과열 → 일봉 ATR로 캡
atr_intraday = min(atr_5min * sqrt(78), atr_daily * 0.6)
```

**데이트레이딩 전략별 캔들 기반 SL (가장 명확한 방법):**

| 전략 | SL 로직 | 트레이더 논리 |
