### 1-A. `TradingScheduler.java` – `scanAuction()` S7 직접 ka10029 호출 제거

**현황**: `scanAuction()` 은 여전히 내부에서 ka10029 를 직접 호출하여 `gapSet` 을 구성한다.
`getS7Candidates("001/101")` 이 이미 ka10029 2~10% 필터 결과를 `candidates:s7:` 에 캐싱하므로 중복 API 소모가 발생한다.

**목표 변경**:
```java
// Before (scanAuction 내부)
KiwoomApiResponses.ExpCntrFluRtUpperResponse gapResp =
    kiwoomApiService.fetchKa10029(...);
gapResp.getItems().stream()
    .filter(item -> { ... f >= 2.0 && f <= 10.0 ... })
    .forEach(gapSet::add);

// After – 캐시된 풀 재사용
Set<String> gapSet = new java.util.HashSet<>();
gapSet.addAll(candidateService.getS7Candidates("001"));
gapSet.addAll(candidateService.getS7Candidates("101"));
```
> ka10030 거래대금 필터(`volSet`)·BidUpper(`bidSet`) 교집합 로직은 그대로 유지.

**변경 파일**: `api-orchestrator/.../scheduler/TradingScheduler.java` (`scanAuction()`)

### 1-B. `TradingScheduler.java` – `scanBigCandle()` S4 `getAllCandidates()` 교체

**현황**: S4 는 surgeSet(거래량급증 ∪ 가격급등) 필터를 거친 뒤 `getAllCandidates()` 를 폴백으로 사용한다.
`getAllCandidates()` 는 구 `candidates:001` + `candidates:101` (ka10029 3~30% 전체) 를 반환하는 방식으로 S4 의미와 거리가 있다.

**목표 변경**:
```java
// surgeSet 이 비어 있을 때 폴백을 S12(거래대금 상위) 풀로 교체
if (surgeSet.isEmpty()) {
    candidates = java.util.stream.Stream.concat(
            candidateService.getS12Candidates("001").stream(),
            candidateService.getS12Candidates("101").stream())
        .distinct().limit(100).collect(Collectors.toList());
} else {
    candidates = java.util.stream.Stream.concat(
            candidateService.getS12Candidates("001").stream(),
            candidateService.getS12Candidates("101").stream())
        .distinct()
        .filter(surgeSet::contains)
        .limit(30).collect(Collectors.toList());
}
```
> S4(장대양봉) 는 거래대금 상위 + 가격/거래량 급증 교집합이 의미적으로 더 정확.

**변경 파일**: `api-orchestrator/.../scheduler/TradingScheduler.java` (`scanBigCandle()`)
