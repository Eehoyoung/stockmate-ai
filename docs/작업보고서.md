## 수행한 작업

### 버그 수정

1. **`analyzer.py`** – Claude 응답 텍스트에서 JSON 블록만 추출하는 `json_start`/`json_end` 슬라이싱 로직 추가
2. **`health_server.py` (websocket-listener + ai-engine)** – `asyncio.Event().wait()` + `runner.cleanup()` 추가로 서버 수명 보장
3. **`ws_client.py`** – `subscribed_set`을 초기 candidates로 채워 spurious UNREG 발송 방지
4. **`queue_worker.py`** – FORCE_CLOSE/DAILY_REPORT 항목이 AI 분석 없이 ai_scored_queue로 직통 전달
5. **`TradingSignalDto.java`** – `signal_time`, `cur_prc` 필드를 `toQueuePayload()` 에 추가
6. **`TradingController.java`** – `/strategy/s2/run` (안내 메시지), `/strategy/s4/run` (장대양봉 스캔) 엔드포인트 추가

### 코드 개선

7. **`http_utils.py` 신규 생성** – `fetch_cntr_strength()` 공통 유틸 모듈 생성 (strategy_2, strategy_6 중복 제거)
8. **`strategy_2_vi_pullback.py`** – 중복 `fetch_cntr_strength` 삭제, `http_utils` 임포트로 전환
9. **`strategy_3_inst_foreign.py`** – 전체 httpx 호출에 `timeout=10.0` + `raise_for_status()` 추가
10. **`strategy_5_program_buy.py`** – 전체 httpx 호출에 `timeout=10.0` + `raise_for_status()` 추가
11. **`strategy_6_theme.py`** – 중복 `fetch_cntr_strength` 삭제, timeout 추가, `raise_for_status()` 추가
12. **`queue_worker.py`** – `_fallback` 상단 임포트로 이동 (인라인 임포트 제거)
13. **`redis_reader.py`** – `push_score_only_queue()`에 try/except + `default=str` 폴백 추가
14. **`kiwoom.js`** – `runStrategy()` 맵에 `s2`, `s4` 추가; 에러 메시지에 허용 범위 안내 포함
15. **`formatter.js`** – `escapeHtml()` 유틸 추가; `ai_reason` 필드에 HTML 이스케이프 적용; `signal_time` 파싱 try/catch
16. **`signals.js`** – 분당 최대 신호 발송 수(`MAX_SIGNALS_PER_MIN`, 기본 10) 제한 로직 추가
17. **`index.js`** – `shutdown()`에서 `bot.stop()`, `closeRedis()` 개별 try/catch 감싸기
18. **`commands.js`** – `/시세`에 6자리 숫자 검증, 등락률 부호 표시, 누적거래량 표시, toLocaleString 포맷 추가
19. **`.gitignore`** – Python venv, __pycache__, logs, Node node_modules 패턴 추가

### 전략 로직 개선 (strategy_runner + scorer)

20. **`strategy_runner.py`** – S4(장대양봉)를 Python 스캐너에 추가 (상위 30개 후보 순차 스캔); S7을 S1보다 먼저 실행 (장전 동시호가 우선 처리)
21. **`scorer.py`** – 전 전략 스코어 수식 세분화:
    - S1: 체결강도 중간 단계(110~130%) 점수 추가; 호가비율 중간 단계(1.3~1.5) 추가; 신호 내 cntr_strength 보너스 추가
    - S2: 체결강도/호가비율 중간 단계 추가
    - S3: 거래량비율 3단계 세분화 (≥3x, ≥2x, ≥1.5x)
    - S4: 체결강도 3단계 세분화; 거래량비율 ≥3x 기준 추가
    - S5: 체결강도/호가비율 상위 단계 기준 강화
    - S6: 신호 내 cntr_strength 값 우선 사용
    - S7: 거래량 순위 3단계 (≤10, ≤20, ≤30)로 세분화
    - 공통: 10~15% 등락률 구간 중간 페널티(-10) 추가

## 미완료 항목 / 추천 후속 작업

### 단기 (P1 – 즉시 적용 권장)

1. **단위 테스트 추가** – `scorer.py`, `analyzer.py`, `redis_reader.py`에 대한 pytest 테스트 작성. 현재 테스트 코드 전무.
2. **`is_dynamic` 필드 타입 통일** – Java DTO는 `Boolean`, Python scorer는 `signal.get("is_dynamic")`으로 truthy check 사용. Java 직렬화 시 `true`/`false` vs `1`/`0` 불일치 가능성 확인 필요.
3. **S3 `continuous_days` 하드코딩** – `strategy_3_inst_foreign.py`의 `scan_inst_foreign()`이 `"continuous_days": 3`을 하드코딩. API 응답에서 실제 연속 일수를 추출하도록 개선 필요.
4. **S5 market 파라미터** – `strategy_5_program_buy.py`의 `scan_program_buy()`가 `market="000"`을 기본값으로 사용하지만, API가 전체 종목(000) 결과를 지원하는지 검증 필요.

### 중기 (P2)

5. **Redis 연결 재시도 로직** – `ai-engine`의 Redis 단절 시 재연결 없이 오류 무한 반복. `aioredis`의 `retry_on_error` 또는 별도 재연결 루프 구현 권장.
6. **Claude API 비용 모니터링** – `claude:daily_calls:{date}` 카운터를 Telegram `/report` 명령에 노출하여 실시간 API 사용량 확인 기능 추가.
7. **신호 중복 TTL 조정** – `isSignalDuplicate()`의 1시간 TTL이 짧을 수 있음. 전략별로 TTL 조정 가능하도록 설정화.
8. **`strategy_runner.py` 동시 실행 제한** – S4 스캔이 순차 실행되어 스캔 간격(60초)을 초과할 위험. `asyncio.gather()` + 동시성 제한 (semaphore) 적용 권장.

### 장기 (P3)

9. **알림 템플릿 분리** – formatter.js의 메시지 형식을 설정 파일로 분리하여 유지보수성 향상.
10. **로그 집계** – 각 모듈의 로그를 중앙화(ELK Stack, Loki 등)하여 전략별 성과 분석 가능하도록 구성.
11. **Mock API 테스트 환경** – `KIWOOM_MODE=mock` 시 실제 API 대신 응답하는 로컬 목서버 구현.
