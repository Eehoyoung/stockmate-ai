## 1. 작업 배경 및 목적

기존 스코어 시스템에 3가지 구조적 결함이 존재해 신호 품질이 저하되고 있었음.

| 원인 | 증상 |
|------|------|
| signal dict 필드 누락 | scorer가 0점 처리 → 유효 신호가 CANCEL |
| WS 미구독 종목 bid_ratio = 0 | 스윙 전략 호가 항목 항상 0점 또는 오채점 |
| WS 미구독 종목 flu_rt = 0 | 과열·하락 페널티 미작동, 등락률 점수 0 |
| API 응답 필드명 불일치 | S3 순매수 항목 항상 0 (25pt 사망) |
| `clean_num()` 음수 부호 제거 | 하락 등락률이 상승으로 오인, 필터 오통과 |

## 6. 답변: 모든 전략이 정상 스코어링되는가?

**예 — 단, 하나의 전제 조건 충족 시.**

| 조건 | 상태 |
|------|------|
| Redis `candidates:s{N}:{market}` 풀이 Java Orchestrator에 의해 적재됨 | Java 기동 필요 |
| WS 구독 종목: tick/hoga/strength Redis 키 존재 | websocket-listener 기동 필요 |
| WS 미구독 스윙 종목: signal 내 필드로 자체 완결 | **이번 작업으로 해결** |
| Kiwoom token 유효 | kiwoom:token Redis 키 필요 |

WS 미구독 스윙 종목(S8~S15)의 경우, 이번 작업 이전에는 scorer가 0점 처리하거나 잘못된 점수를 부여하여 대부분 CANCEL되었음.  
수정 후에는 strategy 파일이 signal에 직접 필요한 데이터를 담아 전달하므로, WS 구독 여부와 무관하게 정상 채점됨.
