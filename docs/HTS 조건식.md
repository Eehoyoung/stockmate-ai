# HTS 조건식 설정 가이드

> 작성일: 2026-03-22
> 대상: 영웅문4 (키움증권 HTS)
> 목적: StockMate AI 신규 전략 S8 · S9 · S13 종목 선별용 조건식 등록

---

## 개요

S8(골든크로스 스윙), S9(눌림목 스윙), S13(박스권 돌파 스윙) 전략은
**전 종목 일봉 API 개별 호출이 비현실적**이므로, 영웅문 HTS 조건검색기에서
조건식을 사전 생성한 뒤 `ka10172`(일반) 또는 `ka10173`(실시간) API로 연동합니다.

### 조건식 등록 후 해야 할 일

조건식을 저장하면 `ka10171`(조건검색 목록조회) API를 호출하여
`cond_nm`(조건식명)과 `cond_id`(조건식 ID)를 확인한 뒤,
각 전략 Python 파일의 `COND_NM` / `COND_ID` 상수에 입력합니다.

| 전략 | 파일 | 변경할 상수 |
|------|------|------------|
| S8 | `ai-engine/strategy_8_golden_cross.py` | `COND_NM`, `COND_ID` |
| S9 | `ai-engine/strategy_9_pullback.py` | `COND_NM`, `COND_ID` |
| S13 | `ai-engine/strategy_13_box_breakout.py` | `COND_NM`, `COND_ID` |

## 영웅문 조건검색기 접근 방법

```
영웅문4 실행
→ 상단 메뉴: [조건검색] 탭 클릭
→ 또는 단축키: [Alt] + [J]
→ 좌측 패널에서 [조건식 추가] 버튼 클릭
→ 조건식 이름 입력 후 각 항목 조건 추가
```

## 전략 S8 — 20일선 골든크로스 스윙

### 조건식명 (권장)
```
SMA_골든크로스_스윙
```

### 조건 설정 순서

#### 1) 이동평균 골든크로스 (필수)

```
[지표 선택]  주가 이동평균 > 주가 이동평균

설정값:
  - 단기 이동평균: 5일
  - 장기 이동평균: 20일
  - 조건: 단기 > 장기 (상향 돌파, 당일 크로스)

체크: ☑ 당일 교차 (전일에는 5일선 ≤ 20일선이어야 함)
```

> **영웅문 경로**: 지표 → 이동평균 → 이동평균 방향/배열 → "골든크로스"

#### 2) 거래량 조건

```
[지표 선택]  거래량 > 이동평균거래량

설정값:
  - 기준: 당일 거래량
  - 비교: 20일 평균 거래량
  - 조건: ≥ 150% (1.5배 이상)
```

> **영웅문 경로**: 지표 → 거래량 → 거래량비율(이동평균대비)

#### 3) RSI 구간 필터

```
[지표 선택]  RSI

설정값:
  - 기간: 14
  - 조건: ≥ 40 (과매도 탈출)
  - 조건: ≤ 65 (과매수 진입 전)
  → 40 ≤ RSI(14) ≤ 65
```

> **영웅문 경로**: 지표 → 모멘텀 → RSI

#### 4) 60일선 위치 확인

```
[지표 선택]  현재가 > 이동평균

설정값:
  - 이동평균: 60일
  - 조건: 현재가 ≥ 60일 이동평균 × 0.95
    (60일선 5% 이내 또는 상단)
```

#### 5) 기본 필터 (제외 조건)

```
[조건 추가]  종목 조건
  ☑ 관리종목 제외
  ☑ 거래정지 종목 제외
  ☑ 우선주 제외
  ☑ ETF 제외

[추가 조건]  등락률
  - 당일 등락률 ≤ 15% (이미 급등한 종목 제외)

[추가 조건]  시가총액
  - 시가총액 ≥ 500억 원
```

### 최종 조건 요약

| 번호 | 조건 | 값 |
|------|------|-----|
| 1 | 5일 이평 골든크로스 20일 이평 | 당일 상향 돌파 |
| 2 | 당일 거래량 / 20일 평균 거래량 | ≥ 150% |
| 3 | RSI(14) | 40 이상 65 이하 |
| 4 | 현재가 ≥ 60일 이평 × 0.95 | — |
| 5 | 관리종목 · ETF · 우선주 | 제외 |
| 6 | 당일 등락률 | ≤ 15% |
| 7 | 시가총액 | ≥ 500억 |

### 조건식 저장 및 ID 확인

```
1. [저장] 버튼 클릭 → 조건식명: SMA_골든크로스_스윙
2. 저장 후 HTS 상단에 조건식 번호(ID) 표시됨
3. 또는 ka10171 API 호출로 cond_id 확인:
   POST /api/dostk/cond  (api-id: ka10171)
   → 응답의 cond_id 값을 strategy_8_golden_cross.py에 입력
```

## 전략 S9 — 눌림목 지지 반등 스윙

### 조건식명 (권장)
```
SMA_눌림목_스윙
```

### 조건 설정 순서

#### 1) 이동평균 정배열 확인 (필수)

```
[지표 선택]  이동평균 배열

설정값:
  - 5일 이동평균 > 20일 이동평균 > 60일 이동평균
  - 조건: 정배열 (3선 모두 순서대로 정렬)
```

> **영웅문 경로**: 지표 → 이동평균 → 이동평균 방향/배열 → "정배열"

#### 2) 눌림 구간 확인 (필수)

```
[지표 선택]  현재가 vs 이동평균

설정값:
  - 현재가 ≥ 5일 이동평균 × 0.97  (5일선 3% 이하 눌림 시작)
  - 현재가 ≤ 5일 이동평균 × 1.02  (5일선 위로 2% 이내)

의미: 현재가가 5일선 -3% ~ +2% 범위 안에 있음 (눌림 구간)
```

#### 3) 당일 반등 신호 확인

```
[지표 선택]  양봉 여부

설정값:
  - 현재가 > 시가 (당일 양봉)

[지표 선택]  거래량 변화
  - 당일 거래량 ≥ 전일 거래량 × 1.2 (거래량 회복 신호)
```

#### 4) 눌림 중 거래량 감소 확인

```
[지표 선택]  거래량 이동평균 비율

설정값:
  - 최근 3일 평균 거래량 / 최근 10일 평균 거래량 ≤ 80%

의미: 조정 기간 동안 거래량이 줄어든 건강한 눌림
```

> **영웅문 경로**: 지표 → 거래량 → 거래량이동평균비율

#### 5) 기본 필터

```
[조건 추가]  종목 조건
  ☑ 관리종목 제외
  ☑ 거래정지 종목 제외
  ☑ ETF 제외

[추가 조건]  시가총액
  - 시가총액 ≥ 500억 원

[추가 조건]  등락률
  - 당일 등락률 > 0% (하락 종목 제외)
```

### 최종 조건 요약

| 번호 | 조건 | 값 |
|------|------|-----|
| 1 | 5일 > 20일 > 60일 이평 | 정배열 |
| 2 | 현재가 ≥ 5일이평 × 0.97 | — |
| 3 | 현재가 ≤ 5일이평 × 1.02 | — |
| 4 | 당일 양봉 | 현재가 > 시가 |
| 5 | 당일 거래량 / 전일 거래량 | ≥ 120% |
| 6 | 최근 3일 평균거래량 / 10일 평균거래량 | ≤ 80% |
| 7 | 관리종목 · ETF | 제외 |
| 8 | 시가총액 | ≥ 500억 |

## 전략 S13 — 거래량 폭발 박스권 돌파 스윙

### 조건식명 (권장)
```
SMA_박스권돌파_스윙
```

### 조건 설정 순서

#### 1) 박스권 형성 확인 (핵심 조건)

```
[지표 선택]  N일 고가/저가 범위

설정값:
  - 최근 15거래일 최고가 - 최근 15거래일 최저가
    / 최근 15거래일 최저가 × 100 ≤ 8%

의미: 최근 15거래일 동안 주가가 8% 이내 박스권에서 횡보
```

> **영웅문 경로**: 지표 → 고가/저가 → N일 고저가범위비율

> ⚠️ 영웅문에서 "N일 가격범위 비율" 직접 조건이 없다면:
> - **대안**: "최근 20일 최고가 대비 현재가 낙폭 ≤ 8%" + "현재가 / 20일 최저가 ≤ 1.08" 두 조건 AND 결합

#### 2) 박스 상단 돌파 확인 (필수)

```
[지표 선택]  신고가 돌파

설정값:
  - 당일 고가 > 최근 15거래일 최고가

의미: 오늘 처음으로 박스 상단을 상향 돌파
```

> **영웅문 경로**: 지표 → 고가/저가 → 신고가돌파 (기간: 15일)

#### 3) 거래량 폭발 확인 (필수)

```
[지표 선택]  거래량 대 이동평균 비율

설정값:
  - 당일 거래량 / 20일 평균 거래량 ≥ 300% (3배 이상)
```

> **영웅문 경로**: 지표 → 거래량 → 거래량이동평균비율 → 20일 300% 이상

#### 4) 장대 양봉 확인

```
[지표 선택]  양봉 조건
  - 현재가 > 시가 (당일 양봉)

[지표 선택]  종가/고가 비율
  - 현재가 / 당일 고가 ≥ 80%

의미: 장대 양봉 형태 (긴 꼬리 없이 강하게 마감)
```

> ⚠️ 종가/고가 비율 조건이 영웅문에서 직접 지원되지 않는 경우:
> - "윗꼬리 비율 ≤ 20%" 조건으로 대체 가능

#### 5) 기본 필터

```
[조건 추가]  종목 조건
  ☑ 관리종목 제외
  ☑ ETF · ETN 제외
  ☑ 거래정지 제외
  ☑ 우선주 제외

[추가 조건]  시가총액
  - 시가총액 ≥ 300억 원

[추가 조건]  등락률 상한
  - 당일 등락률 ≤ 20% (상한가 또는 과도한 급등 제외)

[추가 조건]  거래대금
  - 당일 거래대금 ≥ 5억 원 (유동성 최소 확보)
```

### 최종 조건 요약

| 번호 | 조건 | 값 |
|------|------|-----|
| 1 | 최근 15일 고저가 변동폭 | ≤ 8% |
| 2 | 당일 고가 > 최근 15일 최고가 | 상단 돌파 |
| 3 | 당일 거래량 / 20일 평균 거래량 | ≥ 300% |
| 4 | 당일 양봉 | 현재가 > 시가 |
| 5 | 현재가 / 당일 고가 | ≥ 80% |
| 6 | 관리종목 · ETF · 우선주 | 제외 |
| 7 | 시가총액 | ≥ 300억 |
| 8 | 당일 등락률 | ≤ 20% |
| 9 | 당일 거래대금 | ≥ 5억 |

## 조건식 저장 후 ID 확인 방법

조건식을 저장한 뒤, 시스템에서 `cond_id`를 확인하는 방법은 2가지입니다.

### 방법 1 — HTS 화면에서 직접 확인

```
영웅문4 → 조건검색 화면 좌측 목록에서 저장된 조건식 선택
→ 조건식명 옆에 번호(ID) 표시됨 (예: "15 SMA_골든크로스_스윙")
→ 해당 번호가 cond_id
```

### 방법 2 — ka10171 API 호출로 확인

```http
POST https://api.kiwoom.com/api/dostk/cond
Header: api-id: ka10171
Header: authorization: Bearer {token}
Body: {}
```

응답 예시:
```json
{
  "cond_list": [
    { "cond_id": "15", "cond_nm": "SMA_골든크로스_스윙" },
    { "cond_id": "16", "cond_nm": "SMA_눌림목_스윙" },
    { "cond_id": "17", "cond_nm": "SMA_박스권돌파_스윙" }
  ]
}
```

## 조건식 작성 시 주의사항

### 영웅문 조건 항목 경로 요약

| 조건 종류 | 영웅문 경로 |
|-----------|------------|
| 이동평균 골든크로스 | 지표 → 이동평균 → 이동평균방향/배열 |
| 이동평균 정배열 | 지표 → 이동평균 → 정배열 |
| RSI | 지표 → 모멘텀 → RSI |
| 거래량 이동평균 비율 | 지표 → 거래량 → 거래량이동평균비율 |
| N일 신고가 돌파 | 지표 → 고가/저가 → 신고가돌파 |
| 고저 변동폭 | 지표 → 고가/저가 → N일고저가범위 |
| 시가총액 필터 | 조건 → 종목 → 시가총액 |
| 양봉/음봉 | 조건 → 주가 → 봉형태 |

### 자주 하는 실수

1. **조건식명 오타**: `COND_NM`은 HTS에 저장한 이름과 **정확히 일치**해야 합니다. 공백·대소문자 포함.
2. **조건 AND vs OR**: 영웅문에서 조건 추가 시 기본값이 AND입니다. 명시적으로 확인하세요.
3. **기간 조건 착각**: 거래량 비율의 "기준 기간"이 1일(당일)인지 N일인지 반드시 확인하세요.
4. **실시간 vs 일반**: S8·S9는 하루 1~2회 스캔이면 충분하므로 `ka10172`(일반) 사용 권장. 실시간이 필요하면 `ka10173`(WebSocket) 연동 필요.

## 조건식 검증 방법

조건식 저장 후 **영웅문에서 직접 실행하여 결과를 확인**합니다.

```
1. 조건검색 화면 → 저장한 조건식 선택
2. [검색 실행] 버튼 클릭
3. 결과 종목 리스트 확인:
   - 결과가 0건: 조건이 너무 엄격하거나 현재 시장에 해당 없음 → 조건 완화 검토
   - 결과가 100건 이상: 조건이 너무 느슨함 → 추가 필터 검토
   - 적정: 5~30건 수준이 이상적
4. 종목 몇 개를 수동으로 차트에서 확인하여 전략 의도에 부합하는지 검증
```
