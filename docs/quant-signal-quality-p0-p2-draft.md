# StockMate AI 전략/신호 품질 개선 초안

시니어 퀀트 트레이더 관점에서, P0/P1/P2 중 전략 실행 품질과 신호 품질에 직접 연결되는 작업만 분리해 정리한다. 제품 UX, 배포 절차, 일반 운영 표준화 항목은 제외했다.

## P0

### 목표

- 신호 파이프라인의 병목 지점을 전략별로 수치화한다.
- `후보 생성 -> 규칙 평가 -> AI 평가 -> 최종 발송` 전환율을 일별로 추적 가능하게 만든다.
- “신호가 적다”를 감각이 아니라 계량 지표로 판단하게 만든다.

### 작업 항목

- 전략별 일일 헬스체크 집계 추가
  - 후보 생성수
  - 규칙 점수 통과수
  - AI 통과수
  - Telegram 발송수
  - 전략별 timeout 수
  - 전략별 최종 CANCEL 사유 비중
- 전략 품질 기준선 정의
  - 전략별 최소 일일 샘플수
  - 허용 timeout 비율
  - 허용 CANCEL 비율
  - 최소 기대 손익비
  - 최소 승률 또는 MFE 우위 기준
- timeout 원인 분해 로그 표준화
  - 데이터 대기
  - 외부 API 지연
  - 내부 연산 병목
  - 큐 적체

### 측정 지표

- 전략별 `candidate_count`
- 전략별 `rule_pass_rate`
- 전략별 `ai_pass_rate`
- 전략별 `publish_rate`
- 전략별 `timeout_rate`
- 전략별 `cancel_reason_mix`
- 전략별 `daily_signal_count`

### 기대효과

- 실제 신호 감소 원인이 후보 부족인지, 점수 기준인지, AI veto인지 즉시 구분 가능해진다.
- 전략 튜닝 우선순위를 감으로 정하지 않고 데이터로 정할 수 있다.
- 장중 장애와 전략 품질 저하를 같은 날 안에 발견할 수 있다.

## P1

### 목표

- 장전/장초 핵심 전략의 입력 안정성과 실행 타이밍을 맞춘다.
- 높은 규칙 점수가 최종 발송까지 이어지지 않는 왜곡을 줄인다.
- 전략 의도와 실제 실행 구간을 정렬해 노이즈를 줄인다.

### 작업 항목

- 장전 후보풀 안정화
  - 후보풀 생성 시점을 `07:50~08:20 KST`로 앞당긴다.
  - `08:25 KST` 기준 후보풀 스냅샷을 고정한다.
  - 전략 시작 직전 후보 종목 수와 업종 분포를 기록한다.
- 전략 시간창 재정의
  - 장전형: `S1`, `S7`
  - 장초형: `S3`, `S6`, `S11`
  - 종가형: `S12`
  - 각 전략은 의도된 시간창에서만 평가되게 제한한다.
- 전략별 timeout 분리
  - 공통 timeout 하나로 관리하지 않고 전략별 override를 둔다.
  - timeout 직전 마지막 수행 단계와 경과 시간을 로그에 남긴다.
- 점수 체계 재보정
  - `S8`, `S9`, `S10`, `S12`의 규칙점수 대비 최종 발송률을 재점검한다.
  - 규칙 점수 구간별 최종 판정 변환표를 만든다.
  - 과도한 CANCEL 구간은 임계값 또는 veto 조건을 재조정한다.

### 측정 지표

- 장전 전략의 `fallback_scan_rate`
- `08:25` 시점 후보풀 크기와 업종 분산도
- 전략별 `median_runtime_ms`, `p95_runtime_ms`
- 전략별 `timeout_rate`
- 전략별 규칙점수 구간별 `publish_conversion`
- 전략별 `high_score_cancel_rate`

### 기대효과

- 장전 신호의 재현성이 올라가고, 같은 시장에서도 입력 상태가 덜 흔들린다.
- 느린 전략만 정확히 튜닝할 수 있어 전체 스캐너 효율이 좋아진다.
- 점수는 높은데 실제론 거의 안 나가는 전략 왜곡을 줄일 수 있다.
- 전략별 발송 수가 “시장 탓”인지 “시스템 탓”인지 더 빨리 구분된다.

## P2

### 목표

- 신호 품질을 “발송 여부”가 아니라 “실전 체결 가능성과 사후 성과” 기준으로 확장한다.
- 사용자 체감 품질과 전략 품질 지표를 연결한다.
- 전략 유지/중단 판단을 사후 데이터 기반으로 전환한다.

### 작업 항목

- 장전~장초 유동성 필터 강화
  - 예상 체결량
  - 호가 스프레드
  - 동시호가 참여도
  - 최소 유동성 기준 미달 종목 제거
- 중복 발송 정책 정리
  - 동일 종목
  - 동일 전략
  - 다른 전략
  - 재기동 후 재발송
  - 각 경우별 허용/차단 규칙 명시
- 신호 사후평가 표준화
  - 5분 MFE/MAE
  - 15분 MFE/MAE
  - 종가 기준 성과
  - 손익비 달성 여부
  - 전략별 보유시간 적합성
- 장후 리뷰 자동화
  - 전략별 발송수
  - 미발송수
  - timeout 수
  - 후보풀 부족 횟수
  - 당일 성과 요약

### 측정 지표

- 신호별 `spread_at_entry`
- 신호별 `expected_liquidity_score`
- 전략별 `duplicate_alert_rate`
- 전략별 `5m_mfe`, `15m_mfe`, `close_return`
- 전략별 `5m_mae`, `15m_mae`
- 전략별 `rr_hit_rate`
- 전략별 `post_publish_win_rate`

### 기대효과

- 초보자 기준으로도 실제 체결 가능한 신호 비중이 높아진다.
- 재시작이나 중복 로직 때문에 생기는 발송 왜곡을 줄일 수 있다.
- 어떤 전략이 “잘 맞는 전략”인지, 어떤 전략이 “손익비가 좋은 전략”인지 분리 평가할 수 있다.
- 전략 존치 여부를 감이 아니라 사후 성과 데이터로 판단할 수 있다.

## 우선순위 요약

1. P0에서 전략별 전환율, timeout, CANCEL 사유를 먼저 수치화한다.
2. P1에서 후보풀 안정화와 전략별 시간창/timeout 분리를 적용한다.
3. P1에서 높은 규칙점수 대비 낮은 발송률 전략을 재보정한다.
4. P2에서 유동성, 중복 발송, 사후 성과 평가를 붙여 실전 품질을 높인다.

## 한 줄 결론

지금 단계의 핵심은 전략을 더 늘리는 것이 아니라, 기존 전략이 `같은 입력에서 안정적으로 돌고`, `높은 점수가 실제 발송으로 이어지며`, `발송 후 성과까지 추적되는 상태`를 만드는 것이다.
