# [ka10055] 당일전일체결량요청

특정 종목의 당일 체결량과 전일 체결량을 비교하여 상세 데이터를 조회합니다.

## 1. 기본 정보
- **Method:** POST
- **URL:** `https://api.kiwoom.com/api/dostk/stkinfo`
- **Content-Type:** `application/json;charset=UTF-8`

## 2. Request 명세
### Header
| Element | 한글명 | 필수 | 타입 | 설명 |
| :--- | :--- | :---: | :--- | :--- |
| api-id | TR명 | Y | String | ka10055 |
| authorization | 접근토큰 | Y | String | Bearer {token} |
| cont-yn | 연속조회여부 | N | String | Y: 연속조회, N: 일반조회 |
| next-key | 연속조회키 | N | String | 응답에서 받은 next-key 값 |

### Body
| Element | 한글명 | 필수 | 타입 | 설명 |
| :--- | :--- | :---: | :--- | :--- |
| stk_cd | 종목코드 | Y | String | 6자리 종목코드 |
| tdy_pred | 당일/전일 구분 | N | String | `"1"`: 당일, `"2"`: 전일 |

> S3_INST_FRGN 전략에서 당일/전일 각각 1회씩 총 2회 호출하여 거래량 비율을 직접 계산함.

## 3. Response 명세 (output)

응답 최상위 배열키: **`tdy_pred_cntr_qty`**

| Element | 한글명 | 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| stk_cd | 종목코드 | String | 종목코드 |
| stk_nm | 종목명 | String | 종목 명칭 |
| cur_prc | 현재가 | String | 현재가 |
| flu_prc | 대비 | String | 전일대비 금액 |
| flu_rt | 등락율 | String | 등락율 |
| **cntr_qty** | **체결량** | String | **해당 시간대 체결 수량 (S3에서 합산하여 거래량 비율 계산)** |
| vol | 거래량 | String | 당일 현재 거래량 |
| pre_vol | 전일거래량 | String | 전일 최종 거래량 |
| vol_rt | 거래량비 | String | 전일 거래량 대비 당일 거래량 비율 (직접 계산 대신 이 값 사용 가능) |
| buy_vol | 매수체결량 | String | 당일 매수 체결 합계 |
| sell_vol | 매도체결량 | String | 당일 매도 체결 합계 |

## 4. 전술 사용처

| 전술 | 사용 목적 |
|------|----------|
| S3_INST_FRGN | 동시간 거래량 비율 계산: 당일(`tdy_pred=1`) ÷ 전일(`tdy_pred=2`) |

## 5. 코드 예시 (strategy_3_inst_foreign.py)

```python
# 당일/전일 각각 호출, cntr_qty 합산으로 vol_ratio 계산
today_qty = sum(int(x.get("cntr_qty", 0))
                for x in today.json().get("tdy_pred_cntr_qty", []))
prev_qty  = sum(int(x.get("cntr_qty", 0))
                for x in prev.json().get("tdy_pred_cntr_qty", []))
vol_ratio = today_qty / prev_qty if prev_qty > 0 else 0
```

> 💡 최적화 팁: `vol_rt` 필드가 거래량비를 직접 제공하므로, 추후 호출 2회 → 1회로 줄일 수 있음.
