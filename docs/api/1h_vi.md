# 1h – VI발동/해제 (실시간)

> **전술 사용**: S2 – VI 발동/해제 감지 → 눌림목 감시 큐 등록

**WebSocket URL**: `wss://api.kiwoom.com:10000/api/dostk/websocket`

## 구독 요청 (전종목, item 빈 문자열)

```json
{
  "trnm": "REG",
  "grp_no": "4",
  "refresh": "1",
  "data": [{"item": [""], "type": ["1h"]}]
}
```

## S2 처리 흐름

```
1. 1h 수신
2. 9068 == "1" (발동)  → Redis vi:{stkCd} 에 active 저장
3. 9068 == "2" (해제)  → vi_watch_queue 에 등록
   {
     stk_cd:      9001번
     vi_price:    1221번 (VI 발동가격)
     watch_until: 현재시간 + 600,000ms (10분)
     is_dynamic:  1225번이 "동적" or "동적+정적" 이면 true
   }
4. ViWatchService.processViWatchQueue() (5초마다)
   → 현재가(ws:tick) 와 vi_price 비교
   → 눌림 -1%~-3% 이면 S2 신호 발행
```

## VI 종류별 특징

| 구분 | 발동 조건 | 특징 |
|------|-----------|------|
| 정적 VI | 기준가 대비 ±10% | 단기 급등락 억제 |
| 동적 VI | 직전 체결가 대비 ±3% | 순간 급변동 억제 |
| 동적+정적 | 둘 다 해당 | 강력한 변동성 |

- **동적 VI 해제 후 눌림이 더 강한 반등 기대** → `is_dynamic = true` 시 스코어 +15점
