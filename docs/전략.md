## 1. 설계 원칙

### 1-1. 핵심 철학

| 원칙 | 내용 |
|------|------|
| **수급 우선** | 가격보다 수급(외국인·기관·프로그램) 변화를 선행 신호로 사용 |
| **거래량 필터 필수** | 모든 전략에 거래량 조건을 포함 — 거래량 없는 가격 움직임은 신호로 인정하지 않음 |
| **API 호출 최소화** | 전 종목 순회 방식 금지. 순위 API → 후보 압축 → 개별 확인 3단계 구조 유지 |
| **명확한 손절 기준** | 진입과 동시에 손절가·목표가 결정. 조건 변화 시 자동 청산 |
| **Claude AI 활용** | 규칙 기반 1차 필터 후 Claude API가 뉴스·테마·수급 종합 판단으로 최종 점수 부여 |

### 1-2. 전략 유형 정의

| 유형 | 보유기간 | 진입 시점 | 청산 방식 |
|------|----------|-----------|-----------|
| **단타** | 당일 | 장중 실시간 | 장 마감 전 전량 청산 또는 목표가/손절가 도달 시 |
| **종가매매** | 1~5거래일 | 14:50~15:00 동시호가 | 익일 또는 이후 목표가/손절가 도달 시 |
| **스윙** | 3~10거래일 | 당일 종가 또는 익일 장초 | 목표가·손절가·기간 초과 중 먼저 도달 시 |

### 1-3. Claude AI 점수화 기준 (공통)

모든 전략의 후보군은 Claude API로 최종 스코어링 후 `ai_scored_queue`에 전달된다.

| 점수 항목 | 배점 | 평가 기준 |
|-----------|------|-----------|
| 수급 강도 | 30점 | 기관·외국인 순매수 금액/연속일, 프로그램 매수 여부 |
| 기술적 위치 | 25점 | 이동평균 배열, 신고가 여부, 볼린저밴드 위치 |
| 모멘텀 | 20점 | 체결강도, 거래량 배수, 갭률 |
| 뉴스·테마 | 15점 | 관련 뉴스 유무, 테마 랭킹, 공시 여부 |
| 리스크 | 10점 | 시가총액, 유동성, 상한가 근접 여부 (역점) |

> **진입 기준**: Claude 점수 70점 이상 → 신호 발행
> **보류**: 50~69점 → 관찰 큐 유지
> **제외**: 49점 이하 → 폐기

### S1 — 갭 오픈 + 체결강도 시초가 매수

**유형**: 단타 | **운영 시간**: 8:30~9:05 | **목표**: +4% | **손절**: -2%

#### 3-1-1. 진입 조건 (AND)

| 순서 | 조건 | 기준값 | 데이터 출처 |
|------|------|--------|-------------|
| ① | 예상 갭상승률 | ≥ 3%, ≤ 15% | 0H WebSocket `exp_cntr_pric` / `pred_pre_pric` |
| ② | 체결강도 (최근 5분 평균) | ≥ 130% | ka10046 `cntr_str_tm` 상위 5개 평균 |
| ③ | 호가 매수/매도 잔량 비율 | ≥ 1.5 | 0D WebSocket `total_bid_qty` / `total_ask_qty` |
| ④ | ka10029 갭 상위 50위 이내 | 리스트 포함 | ka10029 `exp_cntr_flu_rt_upper` |
| ⑤ | 전일 일봉 하락폭 | ≤ 3% OR 신고가 돌파 종목 우선 | ka10082 `pred_pre_pric` 비교 |

#### 3-1-2. 제외 조건

- 갭상승률 > 15% (상한가 추격 금지)
- 전일 이미 5% 이상 상승 종목 (연속 급등 제외)
- 시가총액 200억 미만 (유동성 부족)

#### 3-1-3. 점수 산정 공식

```
Score = gap_pct × 0.5 + (cntr_strength - 100) × 0.5
```
- `gap_pct`: 갭률 (3~15)
- `cntr_strength`: 체결강도 (130 이상)
- 상위 5개 종목만 신호 발행

#### 3-1-4. 청산 조건

| 청산 유형 | 조건 |
|-----------|------|
| 목표가 청산 | 진입가 대비 +4% |
| 손절 청산 | 진입가 대비 -2% |
| 시간 청산 | 9:30 이후 미달성 시 청산 검토 |
| 강제 청산 | 체결강도 < 100% 3분 지속 시 |

#### 3-1-5. 실전 주의사항

- 8:30~9:00 동시호가 중 잔량 비율은 실시간 변화 — 9:00 이후 확인 필수
- 갭 3~5% 구간은 갭 메우기 발생 빈도 높음 → 전일 강한 양봉 종목 우선
- 스팩·관리종목·거래정지 이력 종목 제외

### S2 — VI 발동 후 눌림목 재진입

**유형**: 단타 | **운영 시간**: 실시간 (이벤트 기반) | **목표**: +3% | **손절**: -2%

#### 3-2-1. 이벤트 흐름

```
1h WebSocket VI 이벤트 수신
    ↓
VI 발동 당시 거래량 ≥ 직전 5분 평균 × 3배 확인
    ↓
VI 해제 후 10분 이내 눌림 감시 시작
    ↓
눌림 조건 충족 시 진입 신호 발행
```

#### 3-2-2. 진입 조건 (AND, VI 해제 후)

| 순서 | 조건 | 기준값 | 데이터 출처 |
|------|------|--------|-------------|
| ① | VI 유형 | 동적 VI 우선 (vi_type 2 or 3) | 1h WebSocket `vi_type` |
| ② | VI 발동 당시 거래량 배수 | ≥ 직전 5분 평균 × 3 | 1h WebSocket `acc_trde_qty` |
| ③ | 현재가 대비 VI 발동가 눌림폭 | -1% ~ -3% | Redis `ws:tick` 실시간가격 |
| ④ | 호가 매수/매도 잔량 비율 | ≥ 1.3 | 0D WebSocket |
| ⑤ | 체결강도 | ≥ 110% | ka10046 또는 Redis `ws:strength` |
| ⑥ | 감시 유효 시간 | VI 해제 후 10분(600초) 이내 | `watch_until` 타임스탬프 |

#### 3-2-3. 제외 조건

- 정적 VI 단독 발동 (vi_type == 1) — 동적 VI 비동반 시 신뢰도 낮음
- 당일 이미 3회 이상 VI 발동 (과열 종목 제외)
- 눌림폭 > -3% (강한 매도세 전환 가능성)

#### 3-2-4. 점수 산정 공식

```
Score = (1 - abs(pullback_pct + 2) / 2) × 40    # 눌림 깊이 (-2%가 최적)
      + (cntr_strength - 100) × 0.3               # 체결강도 가산
      + (bid_ratio - 1.3) × 20                    # 호가 비율 가산
```

#### 3-2-5. 청산 조건

| 청산 유형 | 조건 |
|-----------|------|
| 목표가 청산 | VI 발동가 기준 +3% (VI 재돌파 기대) |
| 손절 청산 | 진입가 대비 -2% |
| 시간 청산 | 진입 후 15분 미달성 시 재평가 |

#### 3-2-6. 실전 주의사항

- 동일 종목 당일 2회 이상 VI 발동 → 2번째부터 신뢰도 급락
- 14:30 이후 VI 눌림은 당일 청산 시간 부족 → 스킵 권장
- VI 감시 TTL은 `vi_watch_queue`에서 Java(`ViWatchService`)가 관리

### S3 — 기관+외국인 동반 순매수 돌파

**유형**: 단타 | **운영 시간**: 9:30~ | **목표**: +3.5% | **손절**: -2%

#### 3-3-1. 진입 조건 (AND)

| 순서 | 조건 | 기준값 | 데이터 출처 |
|------|------|--------|-------------|
| ① | 외국인·기관 당일 동시 순매수 | smtm_netprps_tp: "1" | ka10063 `opmr_invsr_trde` |
| ② | 외국인 상위 20위 AND 기관 상위 30위 | 동시 해당 | ka10065 |
| ③ | 최근 연속 순매수 | ≥ 3거래일 (ka10131 기준) | ka10131 `orgn_for_cont_trde` |
| ④ | 당일 거래량 / 전일 동시간 거래량 | ≥ 1.5 | ka10055 |
| ⑤ | 현재가 위치 | 5일 MA 위 | ka10080 5분봉 MA 계산 |

#### 3-3-2. 단계별 필터링

```
Step 1: ka10063 → 외국인+기관 동시 순매수 종목 추출 (상위 30개)
Step 2: ka10131 → 연속 3일 이상 순매수 종목 교차 필터
Step 3: ka10055 → 거래량 1.5배 이상 개별 확인
Step 4: ka10080 → 5일 MA 위치 확인
Step 5: 순매수 금액 기준 Top 5 신호 발행
```

#### 3-3-3. 점수 산정 공식

```
Score = net_buy_amt / 1억          # 순매수 금액 정규화
      + continuous_days × 5        # 연속일 가산 (일당 5점)
      + (vol_ratio - 1.5) × 10    # 거래량 초과분 가산
```

#### 3-3-4. 청산 조건

| 청산 유형 | 조건 |
|-----------|------|
| 목표가 청산 | 진입가 대비 +3.5% |
| 손절 청산 | 진입가 대비 -2% |
| 수급 전환 청산 | 외국인 또는 기관 순매도 전환 시 익일 청산 |

#### 3-3-5. 실전 주의사항

- ka10063은 장중 실시간 업데이트 — 너무 이른 시간(9:30 이전) 호출 시 데이터 부족
- 연속일 데이터(ka10131)의 `cont_dt` 필드명은 API 버전에 따라 상이 → 폴백 로직 필수

### S4 — 대형 양봉 + 거래량 급증 추격매수

**유형**: 단타 | **운영 시간**: 10:00~14:30 | **목표**: +4% | **손절**: -2.5%

#### 3-4-1. 진입 조건 (AND)

| 순서 | 조건 | 기준값 | 데이터 출처 |
|------|------|--------|-------------|
| ① | 5분봉 양봉 몸통 비율 | ≥ 70% (몸통/캔들범위) | ka10080 5분봉 |
| ② | 시가 대비 현재가 상승폭 | ≥ 3% | ka10080 `open_pric` / `cur_prc` |
| ③ | 직전 5봉 평균 거래량 대비 | ≥ 5배 | ka10080 `trde_qty` |
| ④ | 체결강도 3분 평균 | ≥ 140% | Redis `ws:strength` 리스트 |
| ⑤ | 5분봉 기준 최근 고가 돌파 | 최근 96봉 고가 이상 | ka10080 연속 조회 |

#### 3-4-2. 캔들 분석 공식

```python
# 몸통 비율
body_ratio = (close - open) / (high - low)  # ≥ 0.7
# 상승폭
gain_pct = (close - open) / open * 100      # ≥ 3.0
# 거래량 배수
vol_ratio = current_vol / avg(prev_5_vol)   # ≥ 5.0
```

#### 3-4-3. 단계별 필터링

```
Step 1: strategy_runner.py의 candidates 리스트에서 개별 확인
Step 2: ka10080 5분봉 → 몸통·거래량 계산
Step 3: Redis ws:strength → 체결강도 확인
Step 4: ka10053 당일상위이탈원 → 이탈 이력 확인 (제외)
Step 5: Score 기준 정렬 후 Top 5
```

#### 3-4-4. 점수 산정 공식

```
Score = gain_pct × 0.4
      + body_ratio × 20
      + vol_ratio × 2
      + (cntr_strength - 100) × 0.3
      + (10 if is_new_high else 0)
```

#### 3-4-5. 청산 조건

| 청산 유형 | 조건 |
|-----------|------|
| 목표가 청산 | 진입가 대비 +4% |
| 손절 청산 | 진입가 대비 -2.5% (추격 진입이므로 넉넉하게) |
| 모멘텀 소멸 청산 | 체결강도 < 100% 3분 지속 시 |

#### 3-4-6. 실전 주의사항

- 10분봉/15분봉 기준으로도 같은 조건 적용 시 노이즈 감소 가능
- 14:30 이후 신규 진입 금지 (당일 청산 시간 부족)
- 상한가 종목 추격 금지 (ka10053 이탈원 확인)

### S5 — 프로그램 순매수 + 외국인 동시 상위

**유형**: 단타 | **운영 시간**: 10:00~14:00 | **목표**: +3% | **손절**: -2%

#### 3-5-1. 진입 조건 (AND)

| 순서 | 조건 | 기준값 | 데이터 출처 |
|------|------|--------|-------------|
| ① | 프로그램 순매수 상위 50위 이내 | 금액 기준 | ka90003 `prm_netprps_upper_50` |
