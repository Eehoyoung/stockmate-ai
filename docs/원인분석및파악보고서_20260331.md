## 원인 1 – Kiwoom API: HTTP 200이지만 에러 바디 반환 (주요 원인)

### 메커니즘

Kiwoom REST API 서버(Spring Boot 기반)는 내부 오류가 발생해도 **HTTP 상태코드를 200으로 고정**하고
응답 바디에 Spring Boot 기본 에러 포맷을 그대로 담아 반환한다.

```
HTTP/1.1 200 OK
Content-Type: application/json

{
  "date": "2026-03-31T01:43:12.553+0000",
  "error": "INTERNAL_SERVER_ERROR",
  "status": 500,
  "message": "예기치 못한 에러가 발생하였습니다"
}
```

정상 응답 예시 (비교):
```json
{
  "return_code": "0",
  "return_msg": "정상",
  "exp_cntr_flu_rt_upper": [ {...}, {...} ]
}
```

### 발생 시점 추정

- 로그 타임스탬프: `2026-03-31T01:43:12.553+0000` → KST 10:43 (장중)
- 가능한 원인: 순간 과부하 / 서버 재시작 / 특정 파라미터 처리 오류

### Python 코드의 무방비 파싱

`strategy_1_gap_opening.py`(및 동일 패턴의 모든 전략 파일):

```python
# 현재 코드 (문제)
resp = await client.post(KIWOOM_BASE_URL + "/api/dostk/rkinfo", ...)
items = resp.json().get("exp_cntr_flu_rt_upper", [])  # 에러 바디에는 이 키가 없음 → []
```

에러 바디 `{"date":..., "error":..., "status":500, "message":...}`에는
`exp_cntr_flu_rt_upper` 키가 없으므로 `.get(..., [])` 가 **빈 리스트를 반환**,
코드는 오류 없이 "후보 0건"으로 처리하고 계속 진행한다.

`raise_for_status()`는 HTTP 상태코드만 체크하므로 **HTTP 200이면 예외를 발생시키지 않는다**.

```python
resp.raise_for_status()  # HTTP 200 → 통과, 에러 바디는 감지 불가
data = resp.json()       # {"date":..., "error":..., "status":500}
```

### Java 코드의 미처리 경로

`KiwoomApiService.java`의 `buildPostMono()`:

```java
.onStatus(HttpStatusCode::is4xxClientError, response -> on4xx(apiId, response))
.bodyToMono(respType)  // HTTP 200이므로 역직렬화 시도
```

- `@JsonIgnoreProperties(ignoreUnknown = true)` 덕분에 역직렬화 자체는 성공
- 하지만 `returnCode = null` (에러 바디에 `return_code` 필드 없음)
- `isSuccess()` 는 `"0".equals(null)` → `false`
- **서비스 코드에서 `isSuccess()` 확인 없이 `getItems()` 직접 호출 시 NPE → 500**

이것이 사용자가 본 `{"error":"INTERNAL_SERVER_ERROR","status":500}` 이
**두 번째 계층(Java api-orchestrator)** 에서도 다시 발생하는 이유다.

## 원인 2 – Claude API 401: 유효하지 않은 API 키

```

  invalid x-api-key
```

`ai-engine/.env`의 `CLAUDE_API_KEY` 값이 만료되었거나 잘못 설정되어 있다.

**영향**: Claude 2차 분석 불가 → `confirm_worker.py`가 Claude를 호출할 때마다 401 → 신호가 ai_scored_queue에 적재되지 않음.
단, `queue_worker.py`의 규칙 기반 1차 스코어링(scorer.py)은 정상 동작한다.

## 권장 수정 사항

### 수정 1 – Python: `return_code` 검증 추가 (`http_utils.py` 공통화)

```python
# http_utils.py에 공통 응답 검증 함수 추가
def _validate_kiwoom_response(data: dict, api_id: str) -> bool:
    """
    Kiwoom API는 HTTP 200이어도 에러 바디를 반환할 수 있다.
    'error' 키 존재 또는 return_code != "0" 이면 False.
    """
    if "error" in data:
        logger.warning("[%s] Kiwoom 에러 바디 수신: %s", api_id, data.get("message", ""))
        return False
    rc = data.get("return_code")
    if rc is not None and rc != "0":
        logger.warning("[%s] return_code=%s msg=%s", api_id, rc, data.get("return_msg", ""))
        return False
    return True
```

각 전략 파일의 파싱 전에 `_validate_kiwoom_response()` 호출:

```python
data = resp.json()
if not _validate_kiwoom_response(data, "ka10029"):
    return []   # 빈 후보 반환, 재시도 없음
items = data.get("exp_cntr_flu_rt_upper", [])
```

### 수정 2 – Java: 응답 바디 에러 감지 (`KiwoomApiService.java`)

`buildPostMono()` 에서 역직렬화 후 `isSuccess()` 확인:

```java
private <T extends BaseResponse> Mono<T> buildPostMono(...) {
    return spec.bodyValue(body)
            .retrieve()
            .onStatus(HttpStatusCode::is4xxClientError, response -> on4xx(apiId, response))
            .bodyToMono(respType)
            .flatMap(resp -> {
                if (resp != null && !resp.isSuccess()) {
                    log.warn("Kiwoom 응답 오류 [{}] code={} msg={}",
                        apiId, resp.getReturnCode(), resp.getReturnMsg());
                    return Mono.error(new KiwoomApiException(
                        "API 오류 [" + apiId + "] " + resp.getReturnMsg()));
                }
                return Mono.justOrEmpty(resp);
            });
}
```

> **주의**: `respType`이 `BaseResponse`를 상속하지 않는 DTO에 대해서는
> 별도 처리 필요 (제네릭 타입 바운드를 `<T extends BaseResponse>`로 제한하거나 오버로드).

## 참고: Kiwoom HTTP 200 + 에러 바디 패턴 구분법

| 구분 | HTTP 상태 | 바디 형식 | 판단 방법 |
|------|-----------|----------|----------|
| 정상 | 200 | `{"return_code":"0", "return_msg":"정상", ...}` | `return_code == "0"` |
| API 오류 | 200 | `{"return_code":"-1", "return_msg":"파라미터 오류"}` | `return_code != "0"` |
| 서버 내부 오류 | 200 | `{"error":"INTERNAL_SERVER_ERROR","status":500,"message":"..."}` | `"error"` 키 존재 |
| Rate Limit | 200 (일부) / 429 | `{"return_code":"1700", ...}` | `return_code == "1700"` |
| 인증 오류 | 401 | `{"return_code":"8005", ...}` | HTTP 4xx → `on4xx()` 처리 |
