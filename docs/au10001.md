# 🔐 au10001: 접근토큰 발급 (OAuth 인증)

[cite_start]키움 REST API 사용을 위한 최초 인증 토큰을 발급받는 API입니다[cite: 10].

### 1. 기본 정보
- [cite_start]**URL:** `POST https://api.kiwoom.com/oauth2/token` [cite: 10]
- [cite_start]**모의투자:** `https://mockapi.kiwoom.com/oauth2/token` [cite: 10]
- [cite_start]**Content-Type:** `application/json;charset=UTF-8` [cite: 10]

### 2. Request 명세
- **Headers:**
    - [cite_start]`api-id(Y, Str)`: `au10001` [cite: 11]
- **Body:**
    - [cite_start]`grant_type(Y, Str)`: `client_credentials` 고정 [cite: 14]
    - [cite_start]`appkey(Y, Str)`: 발급받은 App Key [cite: 14]
    - [cite_start]`secretkey(Y, Str)`: 발급받은 Secret Key [cite: 14]

### 3. Response 명세
- **Body:**
    - [cite_start]`token(Y, Str)`: **접근 토큰** (이후 모든 API의 `authorization` 헤더에 사용) [cite: 18, 22]
    - [cite_start]`token_type(Y, Str)`: 토큰 타입 (주로 `bearer`) [cite: 18, 22]
    - [cite_start]`expires_dt(Y, Str)`: 만료일시 (YYYYMMDDHHMMSS) [cite: 18, 22]
    - [cite_start]`return_code(Y, Int)`: 처리 결과 코드 (0: 정상) [cite: 22]
    - [cite_start]`return_msg(Y, Str)`: 처리 결과 메시지 [cite: 22]

### 💡 활용 가이드
- [cite_start]발급받은 `token`은 이후 다른 API 호출 시 HTTP Header에 `authorization: Bearer {token}` 형태로 포함해야 합니다[cite: 11].
- [cite_start]토큰의 유효 기간을 확인하여 만료 전 재발급 로직을 구현하는 것이 권장됩니다[cite: 18].
