### FIX-5: Kiwoom REST API Rate Limit – 루프 내 무throttle 호출

**배경**: 키움 REST API 초당 약 5회 제한(error code 1700). 기존 코드는 재시도 로직(실패 후 1초 대기)만 있고 선제적 throttle이 없었다.

| 전략 | 루프 대상 | API 호출 수 (최대) | 위험도 |
|------|-----------|-------------------|--------|
| S1_GAP_OPEN | 후보 종목 (최대 200개) | 200 × ka10046 | **높음** |
| S3_INST_FRGN | 상위 30개 × ka10055 × 2 | 60회 연속 | **높음** |
| S6_THEME_LAGGARD | 5 테마 × N 종목 × ka10003 | 최대 150회 | **높음** |
| S5_PROG_FRGN | asyncio.gather 2회 병렬 | 2회 동시 | 낮음 |
| S10_NEW_HIGH | asyncio.gather 2회 병렬 | 2회 동시 | 낮음 |
| S12_CLOSING | asyncio.gather 2회 병렬 | 2회 동시 | 낮음 |

> asyncio.gather로 2개 동시 호출하는 S5/S10/S12는 제한 범위 내이므로 수정하지 않음.

> 기대 승률은 전략 조건의 이론적 합리성을 기반으로 추정. 실거래 백테스트 수치 아님.

### 4.2 기대 수익 시뮬레이션 (1일 기준, 단일 신호)

```
가정: 신호 5건/일, 자본 1,000만원, 신호당 200만원 배분

시나리오 A (보수적, 승률 45%, 목표 +3%, 손절 -2%)
  기대 수익 = 5건 × [(0.45 × +3%) + (0.55 × -2%)] × 200만원
            = 5 × [+1.35% - 1.10%] × 200만원
            = 5 × 0.25% × 200만원 = +2.5만원/일 (연 약 +600만원)

시나리오 B (중립, 승률 55%, 목표 +3.5%, 손절 -2%)
  기대 수익 = 5 × [(0.55 × +3.5%) + (0.45 × -2%)] × 200만원
            = 5 × [+1.925% - 0.9%] × 200만원
            = 5 × 1.025% × 200만원 = +10.25만원/일 (연 약 +2,500만원)
```

### 4.3 현실적 장애 요인

| 요인 | 심각도 | 현재 대응 |
|------|--------|----------|
| **슬리피지** – 시장가 진입 시 실제 체결가 차이 | 높음 | 목표/손절 설정에 미반영 |
| **수수료** – 매수·매도 각 0.015~0.025% | 중간 | 시뮬레이션 미포함 |
| **실주문 미연결** – 현재 시스템은 신호 발송만, 자동 주문 없음 | 매우 높음 | 수동 발주 필요 |
| **API rate limit** – 초당 5회 제한 위반 시 신호 지연 | 높음 | 이번 세션 일부 수정 |
| **갭메우기** – S1 갭상승 후 되돌림 | 높음 | 체결강도 필터로 일부 대응 |
| **테마 소멸** – S6/S11 수급 뒤집힘 | 중간 | 손절 -2~4% 설정 |
| **Claude API 비용** – 일 100회 기본 설정 | 낮음 | MAX_CLAUDE_CALLS_PER_DAY 조정 가능 |

### 4.4 수익 가능 조건

현재 시스템이 실제 수익을 내기 위한 **필수 조건**:

1. **자동 주문 연결** – 현재 Telegram 신호만 발송됨. Kiwoom KOA Studio 또는 OP API 자동 주문 모듈 추가 필요.
2. **슬리피지 + 수수료 반영** – 목표/손절 각각 0.1~0.15% 여유 확보 필요.
3. **백테스트 수행** – 최소 6개월 과거 데이터로 전략별 실제 승률 측정 필요.
4. **포지션 사이징** – 현재 신호에 투자 금액 정보 없음. 별도 리스크 관리 모듈 필요.
5. **장 후 리뷰** – 일 단위 신호 성과 추적(SignalPerformanceScheduler)을 활용한 전략별 성과 모니터링 체계화.

### 4.5 현재 시스템 강점

- **다층 필터링**: 규칙 스코어(scorer.py) → Claude AI 분석(analyzer.py) → 신호 발송의 3단계 필터
- **수급 기반 전략 우위**: S3(외인+기관), S5(프로그램), S11(외인 연속)은 검증된 수급 추종 논리
- **뉴스 연동 리스크 관리**: news_scheduler.py가 부정적 뉴스 탐지 시 매매 중단 권고
- **섹터 과열 감지**: 동일 섹터 과다 신호 시 SECTOR_OVERHEAT 경보
- **경제 캘린더 연동**: 고영향 이벤트 2시간 전 CAUTIOUS 자동 전환
