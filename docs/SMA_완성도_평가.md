## 1. 비즈니스 완성도 — 82점

### 1.1 충족된 영역

#### 핵심 가치 제안 구현 (★★★★★)

- KOSPI·KOSDAQ 전종목 실시간 모니터링 → WebSocket 0B/0H/0D/1h 수신 완전 구현
- 15가지 독립 전략이 데이트레이딩(7개)·스윙(8개)로 커버리지 균형 확보
- 규칙 스코어 + Claude AI 이중 필터링으로 오신호 비용 최소화 설계 반영
- 신호 추천 시스템 포지셔닝 명확: "자동매매 아님, 최종 판단은 사용자"

#### 사용자 제어 경험 (★★★★☆)

- Telegram 26개 명령어: 조회·설정·제어·알림 사이클 완비
- PAUSE/RESUME 인라인 컨펌 플로우로 리스크 상황에서 모바일 즉시 대응 가능
- 자동 브리핑 4종(장시작·오전중간·일일리포트·뉴스알림) → 사용자가 능동적으로 확인하지 않아도 컨텍스트 유지

#### 리스크 관리 가이드 (★★★★☆)

- 전략별 목표가/손절가 명시, 포지션 사이징 테이블(AI 점수 연동), 일별 손실 한도 3% 원칙
- 섹터 과열 3중 제어(종목 쿨다운·섹터 과열·일일 상한)로 신호 남발 방지

#### 외부 컨텍스트 연동 (★★★★☆)

- 30분 주기 AI 뉴스 분석 → CONTINUE/CAUTIOUS/PAUSE 자동 전환
- FOMC·한은·CPI 등 경제 캘린더 2시간 전 자동 경고 및 CAUTIOUS 전환

### 1.2 미충족·갭 영역

#### ❌ Telegram 전략 필터 S8~S15 미지원

현재 `/filter` 명령은 S1~S7만 지정 가능. S8~S15는 `all` 모드에서만 수신된다. 스윙 트레이더가 특정 스윙 전략만 선택 수신하는 UX가 없다.

> **영향**: 신규 스윙 전략 5개(S8·S9·S13·S14·S15) 추가에도 불구하고 사용자 제어 경험이 확장되지 않음.

#### ❌ 신호 메시지 이모지·라벨 S8~S15 미반영

사용설명서 5.3절의 "전략 이모지" 목록이 S8~S15 부재. formatter.js 레벨에서 신규 전략 식별자 처리 여부 미확인.

#### ❌ 백테스팅 / 과거 성과 데이터 없음

전략 15개 모두 진입 조건은 정의되어 있으나 역사적 데이터 기반 승률 검증 결과가 없다. "S14 목표 +7%, 손절 -4%"가 실제 코스피·코스닥 데이터에서 유효한지 근거 자료 부재.

#### ❌ candidates 풀 관리 주체 불명확

S8~S15는 `candidates:001`, `candidates:101` Redis 키를 폴링하나, 이 후보 풀이 어떤 모듈이 언제 어떻게 갱신하는지 사용설명서에 기재되지 않음. 후보 풀이 비어있으면 스윙 전략 전체가 신호 0건.

## 2. 기술적 완성도 — 76점

### 2.1 충족된 영역

#### 모듈 분리 아키텍처 (★★★★★)

- 4개 독립 모듈(websocket-listener·api-orchestrator·ai-engine·telegram-bot) + Redis 중심 비동기 파이프라인
- 모듈 간 직접 호출 없음. 큐 기반 결합 해제로 개별 재시작·장애 격리 가능
- 헬스체크 3개 엔드포인트(:8080/:8081/:8082) + 종합 헬스 API + Telegram `/에러` 명령

#### 기술 지표 라이브러리 (★★★★★)

S14·S15 추가와 함께 독립 지표 모듈 7개 신규 도입:

| 모듈 | 지표 |
|------|------|
| `indicator_rsi.py` | RSI(14) |
| `indicator_macd.py` | MACD(12,26,9), Histogram |
| `indicator_bollinger.py` | Bollinger Bands, %B |
| `indicator_atr.py` | ATR(14), Williams %R |
| `indicator_stochastic.py` | Slow Stochastic(%K, %D) |
| `indicator_volume.py` | MFI(14), VWAP, get_vwap_minute |
| `ma_utils.py` | MA계산, 골든크로스, 눌림목, 박스권 돌파 감지 |

재사용 가능한 지표 계층 분리는 이후 전략 추가 비용을 크게 낮춤.

#### WebSocket 안정성 (★★★★☆)

- GRP 1~4 단독 운영(Java WS 비활성 시) / GRP 1~4 + 5~8 병렬 운영(Java WS 활성 시) 분기 설계
- 재연결 카운터(`monitor:ws_reconnect_count`) Redis 기록, PM2 프로세스 관리

#### 비용 최적화 (★★★★☆)

- 규칙 점수 임계값 미달 → Claude API 미호출 → S4(75점), S1·S7(70점) 등 고기준 전략은 Claude 호출 횟수 자동 절감
- `claude:daily_calls:{date}`, `claude:daily_tokens:{date}` Redis 카운터로 실시간 사용량 추적

## 3. 수익적 완성도 — 68점

### 3.1 충족된 영역

#### 손절/익절 체계 (★★★★☆)

- S14·S15는 ATR 기반 동적 손절(ATR × 2.0)·동적 목표(ATR × 3.5) → 시장 변동성에 맞는 적응형 리스크 관리

#### AI 이중 필터 비용 효율 (★★★★☆)

- 규칙 스코어 미달 신호 → Claude 미호출로 일일 API 비용 통제
- Claude 최종 판단이 action=ENTER + ai_score ≥ 65 미충족 시 발송 차단 → 저품질 신호 사용자 노출 최소화
- Claude가 `adjusted_target_pct`, `adjusted_stop_pct`를 재조정하여 시장 상황 반영

#### 신호 품질 가중 포지션 연동 (★★★☆☆)

- AI 점수 90+→200만원 / 80+→150만원 / 70+→100만원 / 65+→50만원 (1,000만원 기준) 테이블 제시
- 섹터 과열·일일 상한(30건)으로 특정 테마 쏠림 방지

#### 성과 추적 피드백 루프 (★★★★☆)

- WIN/LOSS/EXPIRED 자동 평가(장중 10분 주기) → 전략별 가상 승률·수익률 누적
- `/전략분석` 명령으로 전략별 성과 비교 → 저성과 전략을 사용자가 `/filter`로 제외 가능

### 3.2 미충족·갭 영역

#### ❌ 백테스팅 결과 없음 (가장 큰 약점)

전략 설계는 논리적이지만 코스피·코스닥 과거 데이터 기반 검증 없이 라이브 운용을 시작한다.

| 전략 | 설정 손절 | 설정 목표 | 이론 R:R | 역사적 승률 |
|------|----------|----------|---------|-----------|
| S1 갭상승 | -2.0% | +4.0% | 1:2 | **미검증** |
| S8 골든크로스 | -4.0% | +8.0% | 1:2 | **미검증** |
| S14 과매도반등 | ATR×2 | ATR×3.5 | ~1:1.75 | **미검증** |
| S15 모멘텀동조 | ATR×2 | +10~15% | ~1:2.5+ | **미검증** |

기댓값(EV) = 승률 × 평균이익 - 패율 × 평균손실이 플러스임을 실증적으로 보이지 못하면, 시스템이 "좋은 신호를 잘 발견하는지"가 아니라 "신호를 발견하기는 하는지" 수준에 머문다.

#### ✅ ~~슬리피지·수수료 미반영~~ (해소)

`analyzer.py`에 `SLIP_FEE`(KOSPI 0.35%, KOSDAQ 0.45%) 상수 및 실질 R:R 계산 추가. `formatter.js`에 `_effectiveRR()` 구현으로 Telegram 신호 메시지에 **실질R:R(슬리피지반영)** 표시, R:R < 1.0 시 ⚠️ 경고.

#### ✅ ~~오버나잇(스윙) 갭다운 리스크 미관리~~ (해소)

`OvernightRiskScheduler.java` 신규 구현: 08:30 cron, SENT/OVERNIGHT_HOLD 스윙 신호 조회 → `ws:expected` 장전 예상등락율 우선 확인, 없으면 ka10087(시간외단일가) 폴백 → 등락률 ≤ -3% 시 `OVERNIGHT_RISK_ALERT`를 telegram_queue 발행.

#### ✅ ~~candidates 풀 품질이 수익 결과 직결~~ (해소)

전략 특성별 전용 후보 풀 분리 완료: 스윙 전략(ka10027 소폭상승 0.5~8%), 과매도 전략(ka10027 하락률 3~10%), 신고가 전략(ka10016 52주). 각 전략이 의도한 시장 상황의 종목만 필터링하여 수익 품질 개선.

#### ❌ Claude API 점수 일관성 검증 없음

동일 신호를 Claude에 재전송했을 때 ai_score 편차가 어느 정도인지 측정된 바 없다. 확률적 모델 특성상 동일 입력에도 ±5~10점 변동이 발생할 수 있으며, 65점 임계값 근방의 신호는 당일 통과 여부가 운에 달릴 수 있다.

## 5. 강점 요약 (차별화 포인트)

SMA의 핵심 경쟁력은 "신호 생성 파이프라인 설계의 완성도"에 있다. 아래 세 가지는 동급 개인 트레이딩 시스템 대비 명확한 우위다.

1. **이중 필터 비용 최적화**: 규칙 점수 임계값으로 Claude API 호출을 절감하면서도 최종 신호 품질을 AI 레벨로 유지하는 구조는 운영 비용 대비 신호 품질 비율이 높다.

2. **컨텍스트 인식 매매 제어**: 단순 신호 발생 시스템이 아니라 뉴스 심리·경제 이벤트·섹터 과열을 통합하여 시장 환경에 따라 신호 발행 자체를 제어하는 메타 레이어가 존재한다.

3. **지표 라이브러리 분리 설계**: RSI·MACD·볼린저·ATR·Stochastic·MFI·VWAP·Williams%R을 독립 모듈로 분리했기 때문에 신규 전략 추가 시 지표 재개발 비용이 0에 가깝다. S14·S15가 이 구조의 효과를 이미 증명했다.
